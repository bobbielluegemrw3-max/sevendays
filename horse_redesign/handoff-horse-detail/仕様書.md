## 1. 何が変わったか（情報設計）

初版は「ヒーロー＋サイド → 能力 → 調教 → 戦績 → 検証」の縦積みで、**最重要の日課
「今日の調教」が能力の下に埋もれ**、価値の物語（現在価値・翌日価値・Day7=200）が
ヒーロー/サイド/文中に分散していた。v2 は次の5段構成に再編:

1. **MASTHEAD** — 名前 ＋ レアリティ/タイプ/状態バッジ ＋ 右肩に現在価値の大スタット。
2. **HERO ROW（2カラム・横並び同格）** — 左=ヒーロー馬アート、右=**今日の調教**。
   「見る（馬）」と「やる（調教）」を同じ高さに並べ、日課を一目で促す。
   出品中/結末では右枠が「出品カード / 結末カード」に切り替わる。
3. **VALUE LADDER（価値の階段・全幅）** — `PRICE_TABLE_V1`（Day0=100 … Day6=177.16）
   ＋ Day7走破=200 を**上昇する棒グラフ**で可視化。「現在地（マゼンタ）」「今夜生き残れば/
   走破すれば（シアン/金）」を1本の物語に集約。分散していた価値表示をここへ統合。
4. **LOWER ROW（2カラム）** — 左=状態（COND/FATIGUE）＋能力＋レアリティ凡例、右=戦績。
5. **PROVENANCE** — 検証情報（DNA/シード/世代）。

ブレークポイントはダッシュボードに合わせ **980px**。未満は1カラム・横スクロール禁止。

## 2. 表示してよい数値（データ規範・厳守）

- `HorseDetail` の実フィールドと `PRICE_TABLE_V1`、および Day7 買戻し額 `200` のみ。
- **架空の統計は一切なし。** 価値は `PRICE_TABLE_V1[clamp(current_day,0,6)]`、
  Day7走破報酬は定数 `200`（価格表の外側＝チャンピオン報酬）。
- 馬の絵は既存 `NftHorseArt`（`deriveNftLook(dna_hash, name)`）のみ。差し替え・画像化不可。
- `condition` / `fatigue` は `stat()` で整数表示、バー幅は `pct()`。
- 能力は日本語ラベル（`ABILITY_JA`）＋**絶対スケール**（`/ ABILITY_MAX=100`）で
  馬同士を見比べられる。生キー表示はしない（087監査 #5）。
- 戦績は `history`（race_results 実データ）のみ。天候×馬場×コースは `WEATHER_JA` /
  `TRACK_JA` / `SURFACE_JA` で日本語化。

## 3. 状態モデル（`modeOf`）

`Mode = 'ACTIVE' | 'LISTED' | 'BURNED' | 'DAY7_CLEARED' | 'MEMORIALIZED'`

- `status==='ACTIVE'` かつ `listing==='MANUAL'` → **`LISTED`**（Market Lock）。
  087監査 #1: 手動出品中は「今夜走らない」を明示し、**調教/アイテムUIを出さない**
  （右枠は出品カード＝`/market` への導線）。
- `status==='ACTIVE'` かつ `listing==='SMART'` → `ACTIVE`（調教可）。masthead に
  「出品中 · スマート」バッジ、`dayNote` に「今夜売れたらレース後に引き渡し」を添える。
- それ以外は `status` をそのまま Mode に採用。

バリューラダー/レール/masthead 値/dayNote はすべて `mode` から分岐（下表）:

- ACTIVE(day<6): 現在=`PRICE[day]` → 今夜生き残れば=`PRICE[day+1]`
- ACTIVE(day>=6): 現在=`PRICE[6]` → 今夜走破すれば=`200`＋記念NFT（Day7列が金・「今夜」）
- LISTED: 出品価格=`PRICE[day]`、「今夜は出走しません（価値は凍結）」
- BURNED: masthead=「消滅」、ラダーの `day` 列を斜線（消滅）、以降は未到達
- DAY7_CLEARED: 報酬=`200` 受取中、Day7列=金「走破」
- MEMORIALIZED: 結末=記念NFT、Day7列=金「走破」

以下はロジック・見た目を**変更せずそのまま利用**する:
- `<TrainingForm horseId horseType fatigue />` — 調教3枚＋おすすめ＋加点/疲労は

## 5. CSS Modules の注意（過去の事故点）

- **リテラル子孫 class は `:global()` で受ける。** 本ページでは馬アート下オーバーレイの
  DAY 表示に `<div className="l">` / `<div className="v">` を使うため、CSS 側は
  `.dayBig :global(.l)` / `.dayBig :global(.v)` で受けている。
- 表示切替系ルールは詳細度を1段上げる方針（本ページは状態分岐を JSX 側の class 付け替えで
  行うため詳細度衝突は少ないが、`.miniCond .miniK` のように親子で色を出し分ける箇所は
  詳細度を上げている）。
- 色・フォントは `globals.css` の `:root` トークンのみ（`--cyan / --magenta / --gold /
  --gold-bright / --magenta-soft / --cyan-deep / --font-display / --font-mono` 等）。

## 6. 空・端の状態（破綻しないこと）

- `history.length === 0`（新規馬・未出走）→ 戦績枠は「まだレースを走っていません。
  最初のレースは今夜20:00です。」の空状態を表示。
- `current_day` は `Math.min(7, …)` / `clamp(0,6)` でガード。DAY7_CLEARED/MEMORIALIZED は
  レール7本すべて点灯＋Day7列=金。
- BURNED はヒーローをグレースケール、レール `day` 本目を減光、以降未到達。
