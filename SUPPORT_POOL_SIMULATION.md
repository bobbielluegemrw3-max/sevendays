# サポートプール モンテカルロシミュレーション(Decision 092候補)

> 2026-07-14。両管理者合意済みの改定案「BURN時に10 USDTをプールへ積立 →
> チャンピオン誕生(7晩走破)トリガーで上位7ティアへ計10 USDT支払い」の破綻検証。
> スクリプト: `packages/settlement-engine/scripts/support-pool-sim.mjs`(乱数シード固定・再現可能)

## モデルの忠実度

- 夜間BURN数 = **floor(rate × 出走頭数)**(憲法のfloor則)
- rate = NORMAL基準10.7% ± 2.7ptの一様対称ジッター、器8.0〜13.5%(ADR-012実装 `nightlyBurnRateV2` と同一)
- BURNは年齢(DAY)無差別にランキング下位 → 年齢比例の無作為配分(超幾何抽出)
- **保守的仮定**: チャンピオン1頭につき満額10を支払う(実際はティア未解放分がプールに残るため出金はこれより少ない)。アイテム代拠出(Decision 078)は収入に数えない

## 結果(20,000トライアル、S4=5,000・S6=2,000)

| シナリオ | 不足発生率 | 期間中最低残高(最悪) | 最終残高(中央値) |
|---|---|---|---|
| S1: 超小規模 5頭/晩×90晩 | **93.3%** | −450 | −90 |
| S2: 単発13頭のみ×30晩 | **100%** | −90 | −30 |
| S3: ローンチ小 30頭/晩×180晩 | **0%** | 0(不足なし) | +4,230 |
| S4: 中規模 100頭/晩×365晩 | **0%** | 0 | +33,130 |
| S5: 需要急停止(100/晩×60晩→0) | **0%** | 0 | +5,200 |
| S6: 大規模 1,000頭/晩×365晩 | **0%** | 0 | +354,730 |
| S7: 不安定需要 0〜60頭/晩×180晩 | 0.01% | −60 | +4,230 |

理論値: 馬1頭あたり入金 10×P(burn)≈5.47 / 出金 10×P(champ)≈4.53 → **期待純増 +0.94 USDT/頭**

## 結論

1. **本番規模(30頭/晩以上)では構造的に破綻しない。** 需要急停止(旧経済を殺したシナリオ)でも不足ゼロ。プールは頭数に比例して黒字が積み上がる(=将来の増額原資)
2. **唯一の弱点は極小頭数**(1晩10頭未満)。floor則により `floor(9×0.107)=0` — BURNが一切発生せず、ほぼ全頭がチャンピオンになるため、入金ゼロのまま出金だけ発生する。**これは今のテストネット(13頭)がまさに該当**
3. 対策(両方推奨):
   - **繰越ルール(構造的保証)**: 支払いはプール残高を上限とし、不足分は待ち行列で繰越(残高回復後に古い順で支払い)。ミント担保ゲートと同じ「財布にある分しか約束しない」哲学。これでプールは**いかなる状況でも負にならない**
   - **テストネットのみ**: デモをスムーズにするため、プールに200〜500 USDTのテスト資金をシード(本番は不要 — S3〜S7で実証)

## 実装時の要点(Decision 092を起票する場合)

- 台帳に `PLATFORM_SUPPORT_POOL` 口座(現行の未達分準備金の後継)
- Step 16(PAY_MLM_REWARDS)の中身を差し替え: BURN毎の+10積立 → チャンピオン毎の7ティア遡り支払い(繰越キュー付き)。**37ステップ構成は不変**
- ティア解放条件(Decision 077)・運営ルートチェーンへの自動帰属(Decision 090)は不変
- 弁護士見解: 生存ベース案について「連鎖販売取引・特定負担に該当せず」(口頭・2026-07-13)。チャンピオントリガー版も同性質だが、実装前に一言確認を推奨
