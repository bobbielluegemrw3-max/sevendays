## 0-Z. 【2026-07-22 夜】7アーキタイプ化 — **成功**(新4型を1枚絵方式で追加)

**結論: v5 プラチナ / v6 銅 / v7 チタン / v8 サムライ を追加し、既存3型と合わせて7型。**
Manus 新6型のうち **2型(クリスタルクローム・ホロメッシュ)は不採用** — 実寸で既存の虹クローム(v3)および
互いと見分けがつかなかった(消去法)。

### 0-A の破綻を繰り返さなかった理由

| 0-A(朝・破綻) | 0-Z(夜・成功) |
|---|---|
| 氷・溶岩・星雲 = **その色でしか成立しない素材** | 全型が**金属**(プラチナ/銅/チタン/濃紺甲冑) |
| coat と mane_tail が独立に回り体と脚が割れた | **レイヤーを分けず絵ごと回す**ので原理的に割れない |
| 承認シートの数値で合格判定した | **実寸150pxとカード実画面で目視**してから採否を決めた |

Manus が各型に**シアン版を同梱**したので、回転耐性をその場で検証できた。銅→緑青(ろくしょう)は銅として正しい変化。

### 方式(既存3型とは別経路)

- 新4型は `<arch>_full.png` **1枚のみ**(768px・透過)。`isSingleArch()` で経路が分かれる
- v6/v7/v8 は絵ごと色相回転(12角)。**v5(プラチナ)は銀=彩度ゼロで色相回転が効かない**ため、
  `transform` に **tint モード**(彩度を足して色を乗せ、明度で濃淡)を新設し 12トーンを持たせた。
  これは 0-A の「将来やるなら」に書いた方針(色相固定・明暗と彩度で振る)そのもの
- **頻度は7型とも均等・無条件**。特定型を珍しくすると「銀=レア」の認識が生まれ、廃止したレアリティの裏口復活になる(R1)
- 明度の下限は 0.72。これ以上暗いとカードの黒背景に沈む(チタンは実寸で x1.30 が最適だった)

### 素材の作り方(再現手順)

原画は白背景の1枚絵(1920px・ヒーロー+ターンアラウンドのシート)。以下を実装側で処理した:
1. ヒーロー部分だけ切り出し(v6=上55% / v7=上45% / 他=全体)
2. 白背景を抜く。**画像の縁から連結している白領域だけ**を背景と見なす(馬の中の白ハイライトは残る)。
   1px収縮 + σ0.7 のぼかしで白フリンジとギザつきを同時に消す
3. 本体(最大連結成分)の**足元より完全に下にある小片**を除去(シート残骸。チタンで4片)
4. 本体高を 495px、接地を y=610 に正規化して 768px キャンバスへ(既存3型の実測 476〜517 / 604〜617 に合わせた)

### 併せて直した既存バグ

隠し演出「全身原色」は **DOMの色付き四角**(`.colorSkin`)だった。旧カードはアート枠の中だったので目立たなかったが、
絵を壁紙にした新カードでは**ただの色ブロックとして露出した**。`NftHorseArt` に `colorVariant` を渡して
**キャンバスの画素へ tint を適用**する方式に変更(馬の形にだけ乗る)。

### 既存馬への影響

アーキタイプ選択が3→7に増えるため、**既存馬の見た目は変わる**(dnaHash からの抽選対象が増えるため)。
テストネット中であること・メインネット移行時に経済リセットが入ることから、今が入れ替えの適期。

## 0-A. 【2026-07-22】6アーキタイプ化は**見送り**(本番投入 → 即リバート)

**結論: 新3型(v5 氷晶 / v6 溶岩 / v7 幻影)は作ったが使わない。既存3型のまま。オーナー判断で見送り。**
素材は手元にあり腐らないので、やるならローンチ経路(書面法務・G10・Button配線)が片付いてから。

### 何が起きたか

Manus に新3体を発注しレイヤー分離納品を受領。**検収は合格**(重複0px・再合成完全一致・白背景残留0px・氷の彩度164)。
既存3体に体高/重心を正規化して768px化し、`ARCHES` を6体へ拡張して本番投入(`dd122c0`) → **実画面で破綻したため即リバート(`b240148`)**。

破綻の実際(オーナーのスクリーンショット):
- `Frozen Road`(氷) → 緑に腐食した破片
- `Cosmic Rider`(星雲) → 茶色く濁った体に紫の脚
- `Phantom Falcon`(幻影) → マゼンタの体に茶色の脚

### 原因(構造的なもの・個別調整では直らない)

**色相回転で192ルックを量産する方式は「色相を回しても成立する素材＝金属」専用**に組まれている。
既存3体(金×黒 / 虹クローム / 黒メカ)は金属なので、何度回しても金属のまま。
新3体は**氷・溶岩・星雲＝その色でしか成立しない素材**なので、回せば必ず壊れる。
さらに `coat` と `mane_tail` が独立に回るため(`NftHorseArt.tsx:110-121`)、体と脚が別色に割れる。

### ★実装側の失敗(記録として残す)

私(実装)は承認シートAで**まさにこの緑の氷馬を見ていた**のに、「平均色差56.4 = 12色すべて別物 = 良好」と
**数値で合格判定**し、「馬として成立しているか」を見なかった。オーナーへ「使えます」と言った根拠が数字だけだった。
`CHAMPION_HERO_POSTMORTEM.md` の「主観報告は実測に変換する」の**逆**をやった — 実測を目視判断の代わりにした。
**教訓: 見た目の変更は、本番へ入れる前に実画面のスクリーンショットをオーナーに見せる。** 合成シートは代わりにならない。

### 将来やるなら(レビュー側Claudeの設計判断・2026-07-22)

- **色相は素材固有の値に固定**(氷=シアン / 溶岩=橙赤 / 星雲=紫)し、**明るさ・彩度でバリエーションを作る**
  (薄氷→濃氷、熾火→業火)。`NftHorseArt.tsx` の `transform()` は現在 rot/mono/desat のみで**明度モードが無い**ため実装追加が必要
- **頻度は既存3型と均等・無条件にする。** 珍しく/条件付き/プレミアムにした瞬間、廃止したレアリティを裏口から復活させることになる(R1・射幸性)
- 目的の再定義: 「3種→6種という数」が目的なら割に合わない。**「金属だけの世界に非金属の質感を足す」**が目的なら6型化しかなく、その場合は上記の方式で作る価値がある

### 資産の所在

- 納品ZIP: `nft_v5v6v7_layers.zip`(リポジトリ直下・未追跡)。原画フラット6枚は `新しい馬のモデル.zip`
- 発注文とレイヤー12枚(768px正規化済み)は **`dd122c0` の中に残っている**。復元は `git show dd122c0` から取り出すだけ
- レビュー側の事実誤認(訂正済み・蒸し返さない): 「氷や星雲に固定の金アクセントが合わない」→ **v5/v7 は accents レイヤーが空**(不透明率0.0%)。金リングを持つのは v6 のみで、それは意図された意匠
