## 0. 位置づけ(不変の原則)

1. **「備え、干渉しない」(R1)**: アイテムはレース内部の確率(BURN率・抽選・他馬)に一切触れない。
   自分の馬を「育てる」「備えさせる」だけ
2. **決定論・公開・検証可能**: 全効果は公開レンジ+シード決定論。運営裁量の効果変更は構造的に不可能
3. **V2スコア式は閉じている(Decision 101)**: `score = total_value + condition_prep(±4) + luck(±3)`。
   `item_modifier` のような専用項は**作らない**。アイテムの作用面は次の2つだけ:
   - **TRAINING系** → `total_value`(調教ロールに合流・ソフトキャップ85の法則に服する)
   - **RACE系** → `condition_prep_modifier`(±4の器・-1bで実装済み)
4. **正直表現**: 「外れたら下がる」を明記して売る。優良誤認になる表現はしない

## 1. 2分類の設計法則

### 1.1 TRAINING系 — 「化けさせる」課金(総合値直効き)

- **使い方**: 調教確定と同時に1個添付(TrainingFormV2に統合)。確定即最終(Decision 107)に
  従い、アイテムもロールと同時に確定・取消不可
- **効果**: 確定ロールの合計デルタに加算(または保険として下振れを消す)。
  **ソフトキャップ85(超過分半減)・上限100はアイテムにも適用**(嘘の数字を見せない)
- **下振れなし〜小**(Decision 104): 無料調教がギャンブル、課金は「確実性を買う」
- **上限: 1頭1サイクル1個**。スタック不可

### 1.2 RACE系 — 予報(70%)を読む「備え」(±4の器への合流)

- **使い方**: レースのスナップショット凍結前まで・1頭1レース1個。凍結前のみ取消可(在庫に戻る)
- **効果の法則(置換方式)**: 素の備え = 天候適性(±2)+馬場適性(±2)の合成(公開適性表・実装済み)。
  RACE系アイテムは**特定の条件グループに備える**:
  - **的中**(実際の条件が備えたグループ): その軸の適性値を**上限側へ置換** `axis := max(axis, +tier)`
  - **外れ**: その軸を**下限側へ置換** `axis := min(axis, −penalty)`
  - 合成後は従来どおり ±4 にクランプ
- **スイングの数学**(シム確定値との整合): 苦手条件(軸−2)に強備え(+2)が的中すると**+4/軸**、
  両軸で最大**+8**。逆に得意条件(+2)で外すと**−4/軸**。これが Decision 101/104 の
  「的中+4〜+8相当・外れ−7〜−3」の実体 — 器(±4)は不変で、**スイングとして**実現する。
  最終数値は実エンジンRTPシム突合でこの分布を再現するよう調整(Decision 104の指示どおり)
- **判定は実際の条件**(予報ではない)。予報70%を「読んで」備えるのがプレイヤーの腕

## 4. ルール・会計(旧仕様の踏襲部分)

- **会計(Decision 078踏襲・変更なし)**: 購入 = `ITEM_PURCHASE`(USER_AVAILABLE→ITEM_CLEARING)。
  使用したレースで馬がBurn → 単価全額 `ITEM_SUPPORT_FUNDING`(→MLM_RESERVE)/
  生存 → `ITEM_REVENUE_SETTLEMENT`(→OPERATING_RESERVE)。TRAINING系は「添付した調教が
  向いていたレース」で同じ規則を適用。未使用在庫はクリアリング滞留(繰延収益)
- **ギフト(Decision 079踏襲)**: `user_transfers` 機構・メール宛・20回/日。`giftable` フラグ維持
- **スナップショット**: 使用アイテムと適用後の値を凍結(リプレイ検証可能・従来どおり)
- Burn率≈10.7%/レース → 売上の約1割が自動的にサポート財源(旧設計の性質を維持)

## 6. ★オーナー判断が必要な点(承認時に○×を)

1. **カタログ表(§2)の品目・名前・価格・効果レンジ** — レンジの最終数値はRTPシム突合で
   ±調整あり(スイング+4〜+8/−7〜−3の再現が基準)
2. **Revenge BuffのV2廃止**(§5)— 弔い=非売ドロップ3種に一本化
3. **アイテム天候(設定1〜6)の廃止**(§5)
4. **非売3種のギフト可否** — 旧Decision 079では「可」だったが、V2でも可とするか
   (可にすると「最強の弔いは買えない」の主張が弱まる — 前回はオーナー了解済み)
5. **コース(TURF/DIRT)の扱い** — Decision 082で「アイテム効果にのみ影響」と定義済みだが、
   **v2.0カタログでは未使用とし、将来の拡張軸として温存する提案**(的中判定を3軸にすると
   初心者の学習コストが跳ねる)
6. **TRAINING系の上限**(1頭1サイクル1個・スタック不可)と **RACE系の上限**(1頭1レース1個)
