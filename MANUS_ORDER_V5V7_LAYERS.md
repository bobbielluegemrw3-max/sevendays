## 0. 状況（なぜ再納品が要るのか）

2026-07-21 に受領した `新しい馬のモデル.zip`（v5 氷晶 / v6 溶岩 / v7 星霜、各2色・計6枚）は**デザインとしては採用**する。ただし **1920×1920 の RGB（白背景・レイヤーなし）** のため、現行エンジンにそのままは載らない。

現行エンジン（`apps/web/components/NftHorseArt.tsx`）は、

- **4レイヤー（`coat` / `mane_tail` / `accents` / `eye_glow`）を真HSVで色相回転して合成**し、
- 1アーキタイプあたり **192ルック**（ボディ12角度 × たてがみ16バリアント）を dnaHash から決定論的に選ぶ

という作りになっている。したがって必要なのは**新規の絵ではなく、既に採用したその3体のレイヤー分離版**である。

## 1. 納品物（v5 / v6 / v7 の3体ぶん）

### 1.1 レイヤーPNG（1体あたり4枚・計12枚）

| ファイル名 | 中身 |
|---|---|
| `nft_v5_coat.png` / `nft_v6_coat.png` / `nft_v7_coat.png` | 本体（胴・脚・頭・首） |
| `nft_v5_mane_tail.png` ほか | たてがみ・尾（＋§2の連動要素） |

**必須仕様**

- **2048 × 2048 px**
- **PNG・RGBA（アルファ必須）**。★今回の納品は白背景の RGB だった。**背景は完全透過**にすること
- **排他分離**: 4レイヤーは互いに重複0px。すべて合成すると元のフラット原画と**ピクセル一致**すること
- 各レイヤーは元画像の RGB をそのまま保持（マスクを切り出すだけ。塗り直し・再生成をしない）
- **白いフリンジ（縁の白ハロー）を残さないこと**。白背景から切り抜く工程で最も出やすい

※ ヒーロー画像などレイヤー合成を使わない箇所で使用する。

### 1.3 メタデータ `layers_meta.json`

## 2. レイヤーの振り分け規則（★今回いちばん重要）

現行エンジンは **coat と mane_tail をそれぞれ独立に色相回転**し、**accents と eye_glow は固定**で描く。つまりレイヤーは「部位」ではなく **「色をどう動かすか」** で決める。

| レイヤー | 規則 |
|---|---|
| `coat` | **個体ごとに色相が回る本体**。胴・脚・頭・首の表面 |
| `mane_tail` | **たてがみと同じ色で動いてほしいもの全て**。たてがみ・尾に加えて、**v6 の溶岩の亀裂**、**v7 の星雲の光筋・星の粒**のように「たてがみと同系色で発光している要素」はここに入れる（位置が胴体上でも構わない） |
| `accents` | **色を動かさない固定パーツ**。金具・装甲板・ハーネス・関節のリング。v5 の銀のフレーム、v6 の金の帯はここ |
| `eye_glow` | 目の発光のみ（固定） |

### 2.1 スピード線・飛散粒子は**納品から除外**すること

3体とも背景に速度線や粒子（氷片・火の粉・星屑）が描かれているが、**これらは含めないでほしい**。理由:

- ゲーム内では馬アートを小さな枠（136〜200px）に収めて描画するため、線が枠外で切れて破綻する
- 現行の v2/v3/v4 には無い要素で、6体並んだときに統一が崩れる

**馬の身体から離れて浮いている粒子・線はすべて透過**にし、身体に接している炎・氷・星雲（たてがみ/尾の延長として自然なもの）は `mane_tail` に含める、という切り分けでお願いしたい。

### 2.2 彩度について（前回の教訓）

**グレースケールに近い塗りは避けてほしい。** 色相回転は「彩度のある色」にしか効かないため、v5（氷）のように白に寄った塗りだと、個体差の色が乗らず全頭ほぼ同じ見た目になる。**氷は淡青ではなく、彩度を持った青**で塗った状態を基準としてほしい。

### 2.3 色違い版（`*_cyan.png`）について

**不要**。色のバリエーションはエンジン側が dnaHash から生成する（1体につき192通り）。納品は**基準色1つ**でよい。

## 3. ポーズ・構図の規格

- **既存3体と同じ枠**に収まること（キャンバス2048正方・馬は右向き・footprint と重心位置を v2/v3/v4 に合わせる = `registered: true`）
- 今回のフラット原画の構図がすでに近いので、**同じ絵の再レンダリング/切り出し**で構わない。新しいポーズは不要

---

## 4. 検収（受け取り側で毎回再計測する）

前回同様、こちらで独立に再計測してから採用を確定する。過去2回の納品で毎回実バグを検出しているため、以下は**必ず通ること**:

1. 4レイヤーの**重複が0px**
2. 再合成が**フラット原画とピクセル一致**
3. 全レイヤーに**アルファがあり、白背景の残留が無い**
4. **縁の白フリンジが無い**（暗背景に置いて目視 + 端画素のサンプリング）
5. `layer_opaque_pct` が妥当（`eye_glow` が極小・`coat` が最大、など前回と同傾向）
