## 1. /account（最も大きく変わるページ）
情報設計を「設定とあなたの記録のハブ」として再構成:

1. **アイデンティティ帯 `.profileCard`** — 大アバター（`avatarHue(me.email)` の hue グラデ／未設定は `W`）＋ メール（未設定は `.vUnset`）＋ `登録日` と `.dayPill`（プレイN日目）＋ `ID`。
2. **記録＋設定 `.recordGrid`** — PC（≥900px）で2カラム、モバイルで1カラム。
   - 記録 `.sec.cy`: 5枚の `Link` タイル `.statCell`（`出走中 / 出品中 / チャンピオン / 消滅 / 割当待ち予約`）。各タイルに `.statGlyph`（◈ ↗ ◆ ✕ ⟳）・`.statV`（数）・`.statK`（ラベル）。色は `.stCyan/.stWarn/.stGold/.stMag`。値は `AccountStats` そのまま。
   - 設定 `.sec`: `TradeAutoTile` と `PwaSetupTile`（**それぞれ dashboard.module.css の意匠のまま**。account 側では触らない）。
3. **連携 `.sec` + `.linkBody`** — 中身は既存 `AccountLinking`。素の `table/button/code/.badge/.stack/h3` を **`account.module.css` の `:global()` で意匠化**。
4. **サポート `.sec`** — `.supportGhost`（使い方）＋ `.supportBtn`（問い合わせ）。

### 算出
- `emailUnset = me.email.endsWith('@user.sevendays')`
- `days = max(1, floor((now - created_at)/86400000) + 1)`
- 統計は `page.tsx` が算出済み（racing=ACTIVE且つlisting≠MANUAL / listed=MANUAL / champions=DAY7_CLEARED+MEMORIALIZED / burned=BURNED / pendingReservations=PENDING_ASSIGNMENT）。

### `:global()` が必須の箇所（過去の事故ポイント）
CSS Modules は子孫セレクタ内のリテラルclassもハッシュ化する。`AccountLinking` が出力するのは **グローバルclass/要素**（`.stack`, `.badge`, `.muted`, `.ok`, `.error`, `table/tr/td/button/code/h3/p`）なので、必ず `:global()` で受ける。すべて `.linkBody` を前置して詳細度を globals.css のベース規則より1段上げ、確実に上書きする（例 `.linkBody :global(button.secondary)`）。`.linkBody` を外すと globals.css の素の table/button が復活する。
