> 作成: 2026-07-06 / 宛先: 日本の弁護士(ゲーム・暗号資産・特商法分野を想定)
> 目的: 本サービスの**実質**を評価いただくための事実整理。マーケティング資料ではなく、不利になり得る事情も含めて記載する。

## 1. サービス概要(事実)

- 競走馬を模したデジタル資産(以下「馬」)を **102 USDT** で購入し、毎日20:00(GMT+8)の自動レースに7日間参加するゲーム。日本語UIあり・グローバル向け。
- レース結果は**決定論的アルゴリズム**(commit-reveal方式の乱数・全結果リプレイ検証可能)。運営の裁量で結果は変わらない。
- 毎日、参加馬の **10.7%〜11.9%**(経済状態による)が「Burn」(敗退・馬の喪失=ユーザーの負け)。
- 7日間生き残った馬は **200 USDT の買戻し**(28.57 USDT×7回の分割払い)+記念NFT(Polygon・ERC-721)を受ける。
- 生存馬は日次の価格ラダー(Day0=100 → Day6=177.16)で**システムが自動的にP2P売却**(Smart Marketplace)することがある。売却益はユーザーに帰属(売り手手数料2%)。
- **カストディアル型**: USDTの入金は運営管理のアドレスへ、残高・馬の保有はDB台帳(複式簿記・不変制約)で管理。出金は運営ホットウォレットから(大口は2名承認)。ユーザーが自己管理するのは記念NFTのみ。

## 2. 資金の流れ(図)

```
ユーザー入金 (USDT / Polygon PoS)
      │  カストディアル(運営HDアドレス → DB台帳残高)
      ▼
馬の購入 102 USDT
      │
      ├─ 本体 100.00 ──┬─ 93.60  買戻し準備金 (PLATFORM_BUYBACK_RESERVE)
      │                ├─  5.40  サポート準備金 (PLATFORM_MLM_RESERVE)   ←★本メモ§3
      │                ├─  0.70  運営準備金
      │                └─  0.30  緊急準備金
      └─ 手数料 2.00 ──┬─  1.00  運営準備金
                       └─  1.00  買戻しバッファ

日次レース(20:00 自動)
      │
      ├─ Burn(敗退 10.7〜11.9%/日) ─→ サポート手当 最大10 USDT を
      │                               サポート準備金から配置ツリー上位10段へ(各1 USDT・条件付き)
      │                               ※支払われない分は準備金に滞留(誰にも配当しない)
      │
      ├─ 生存 ─→ 価格ラダーに沿いシステムが自動でP2P売却することがある
      │          (買い手の支払 → 売り手ユーザーへ。プラットフォーム手数料2%=売り手負担)
      │
      └─ Day7到達 ─→ 買戻し 200 USDT(28.57×7 分割)を買戻し準備金から支払い+記念NFT
```

- 配分比率は「期待負債」に基づく設計: Day7到達率(設計値)46.8%×200=93.60、Burn率53.2%×10=5.32(積立5.40)。
- 準備金は目的別に拘束(台帳の取引タイプで強制)。準備金間の流用は2名承認の緊急復旧手続のみ。
- **新規購入が停止しても、既発行の馬に対する買戻し・サポート手当は積立済み準備金から支払可能**(設計上、後順位者の出捐に依存しない)。

## 3. サポート手当(Burn Recovery Support)— MLM様の多段階制度

詳細仕様は `MLM_REVISION.md`。要点:

- **趣旨(運営の位置付け)**: 紹介行為への対価ではなく、ゲーム敗退者(Burn)の負け分(本来は運営取り分)を原資に、**敗退者の復帰を支援する組織への手当**。
- 発生条件は**Burnイベントのみ**。勧誘・紹介・登録では1円も発生しない。
- 支払先は「配置ツリー」の上位**7ティア**(T1=3 / T2=2 / T3〜T7=各1 USDT)・**総額固定10 USDT/Burn**。
- 受取には各上位者ごとに条件: (a) アカウント有効 (b) **自分が直接紹介したユーザーたちが現在保有する稼働馬の合計価値**に応じたティア解放(T2=3,001 USDT以上〜T7=70,001 USDT以上。毎日20:00に時点評価・維持できなければ自動降格。=実際のゲーム参加が伴わない勧誘は無価値)。
- 条件を満たさないティアの分は**誰にも配当せず準備金に滞留**。
- 紹介関係・配置は一度確定したら変更不可(管理者の例外処理のみ・監査ログ)。

**該当性評価に有利と考える事情**(運営の主張):
1. 原資は購入時の固定積立(5.40/頭)による準備金 — 後順位加入者の出捐を直接配当する構造ではない
2. トリガーはゲームイベント(Burn)であり勧誘行為ではない
3. 総額固定・10段限定・未達分は滞留(組織拡大で支払総額は増えない)
4. 馬の購入は7日間のレース参加権+買戻し到達可能性を持つ**ゲーム資産の取得**であり、配当権の加入料ではない

**不利に評価され得る事情**(正直な記載):
1. 特定負担(102 USDTの購入)+自分の組織の下位のゲーム参加に紐づく利益、という外形は存在する
2. 深い組織ほど(配下のBurn件数が増え)受取機会は増える — 組織拡大の動機は現実に生じる
3. 段数解放条件が「直紹介者の購入ボリューム」であるため、**間接的には紹介先の購入を促す動機**になる

## 4. 検討をお願いしたい論点

1. **無限連鎖講防止法**: §3の構造は無限連鎖講に該当するか。「後順位者の出捐を先順位者に配当する構造ではない(積立済み準備金からの定額・ゲームイベントトリガー)」という反論の有効性。
2. **特定商取引法(連鎖販売取引)**: 該当可能性。該当する場合に必要な義務対応(勧誘時の法定説明・広告表示・概要書面/契約書面・クーリングオフ・禁止行為)の具体的範囲と、グローバルサービスで日本居住者を含む場合の実務。
3. **金融規制**: (a) カストディアルでUSDT(ステーブルコイン)を預かり・出金する行為と資金決済法(暗号資産交換業/電子決済手段等取引業/資金移動業)該当性 (b) 「購入→自動運用→自動利確」の構造と金商法上の集団投資スキーム(みなし有価証券)該当性。※現状ユーザーは売買裁量を持たない(全自動)。裁量付与(手動出品)の追加を検討中 — どちらが安全か。
4. **賭博罪**: 実質金銭を賭け、偶然性(決定論だがユーザーから見て不確定)の結果で得喪が決まる構造への評価。「一人awaiting胴元(運営)対参加者」型での整理。
5. **景品表示法・広告規制**: §5の表現ガイドラインの十分性。
6. **KYC/AML**: 上記該当性判断に応じた要否(現状未実装・仕様未確定事項E18)。
7. **準拠法・利用規約**: 運営主体の設置法域の選択も含めた全体アドバイス。

## 5. マーケティング表現ガイドライン(社内規範・策定済み)

GPTによる一般論レビュー(2026-07-06)でも「**最大のリスクは制度そのものより訴求表現**」と指摘。以下を全チャネル(LP・UI・通知・SNS)の規範とする:

- **禁止**: 「儲かる」「稼げる」「不労所得」「簡単に稼げる」「紹介すると稼げます」「保証された利回り」「リスクなし」「必ず」/ gambling・casino・jackpot・guaranteed profit・risk-free(既存のDaily Derby演出コピー規範と統合)
- **名称**: 「MLM報酬」「コミッション」「紹介報酬」と呼ばない。ユーザー向け正式名称は「**サポート手当(Burn Recovery Support)**」(代替候補: Community Support Allocation / Recovery Support Distribution)。※名称変更だけで実態評価は変わらない前提で、実態(§3)と一致する説明を行う
- **明文化(利用規約・LP・仕様書)**: ①紹介だけでは手当は発生しない ②発生条件はBurnイベントのみ ③総額固定・保証なし
- 訴求の主軸は「ゲームプレイ・馬を育てる・コミュニティ形成」。Burn時の支援制度は**ゲームエコノミーの一部**として説明する
- 既知の表示課題は `PRELAUNCH_COPY_RISKS.md`(R1: 投資商品に見えるリスク / R2: 7日間物語のズレ)に記録済み — 本メモと一体でレビュー対象
