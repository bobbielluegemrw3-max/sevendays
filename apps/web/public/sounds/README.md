## 設計方針(2026-07-21・レビュー側判定)

音は一枚岩ではなく2種類に割れる。この区別が全判断の土台。

| 種別 | 例 | アート依存度 | 扱い |
|---|---|---|---|
| **イベント音**(情報) | 生存の合図、BURNの衝撃、幕の切替 | 低い | 足してよい |
| **ベッド音**(雰囲気・劇伴) | 群衆、緊張を煽る弦、競り合いのSE | 高い | 7/10 ドット走行と同じ轍 |

- **競り合いを煽る音は足さない。** 中間順位のデータは存在しないので、それは
  視覚で排除したフィクションを聴覚で再導入することになる
- **収支のプラス/マイナスで音を変えない。** パチンコの当たり音そのものであり
  R1レッドライン(射幸性を訴求しない)に触れる。価値は数字が語り、音は中立に保つ
- **音が無くても成立することが前提。** 多数がミュートで視聴する。音は加点であって前提ではない
- 帯レースの最良の音は、群衆を引いて作る**沈黙**である

## 2026-07-21 追加の4本(オーナー支給・結線済み)

**きっかけ**: 生存 / DAY7チャンピオン / BURN後の形見ドロップ の3つが、すべて
`own-good.mp3` の同一音だった(オーナー指摘)。感情が違うので分けた。

| 出来事 | 感情 | 音 |
|---|---|---|
| 生存 | 安堵「今夜も生き延びた」 | own-good(据え置き) |
| DAY7チャンピオン | 祝祭 | **champion.mp3** |
| BURN後の形見ドロップ | 慰め | **memorial.mp3** |
| 決算で馬が出ていく | 方向のみ | **settle-out.mp3** |
| 決算で馬が入ってくる | 方向のみ | **settle-in.mp3** |

### 変換パラメータ(原本 = `新音源/` の 48kHz/24bit WAV)

```
champion   ← チャンピオン馬の音源差し替え.wav
             -t 2.1, +1.25dB, afade(out,1.85,0.25), alimiter=0.89
             ※原本は5.26秒だが実音は1.89秒で、末尾3.37秒は無音。切っても中身は無傷。
               審判オーバーレイの表示は3.2秒なので、2.1秒に収める必要がある
memorial   ← メモリアル.wav         volume=+1.59dB, alimiter=0.89
settle-out ← 厩舎から出ていく.wav   -t 2.4, +7.18dB, afade(out,1.8,0.6), alimiter=0.85
settle-in  ← 厩舎に入っていく.wav   -t 2.4, +2.88dB, afade(out,1.8,0.6), alimiter=0.85
        いずれも -ar 44100 -b:a 128k
```

**尺を詰めた理由**: 原本は 5.22s / 4.00s だが、決算幕は馬を **約3.3秒ごと**に
1頭ずつ出す(`settlementFrame`: 10秒 ÷ 頭数)。原本のままだと次の馬の音で
途中から切られる(`playOneShot` は同じ audio 要素を `currentTime=0` で鳴らし直す
ため、重ならずに **打ち切られる**)。2.4秒 + 末尾0.6秒フェードなら、
3頭出る夜でも毎回鳴り切る。

**ゲイン合わせの理由**: 原本の実測は −12.9 / −14.6 / **−25.2** / −20.9 LUFS で、
「出ていく」と「入っていく」に **4.3dB の差**があった。同じ意味の対なので
揃える必要がある。

### 結果として作った音量の階層(実測 LUFS)

```
own-burn    −6.2   ← 最前景。馬を失った衝撃
own-good    −9.3
champion    −9.8   ← own-good と同格。ただし専用音なので格が立つ
memorial   −13.5   ← 一段下。慰めは主張しない
settle-out −19.4   ← 最も下。静かな決算の下で「方向が分かる」以上には主張させない
settle-in  −19.1
```

★出ていく/入ってくる の**方向**だけを鳴らすこと。**収支のプラス/マイナスで音を
変えてはいけない** — それはパチンコの当たり音そのもので、R1レッドライン
(射幸性を訴求しない = AML/KYC不要見解の土台)に触れる。価値は数字が語る。
同じ理由で champion は「換金(200 USDT)の音」ではなく「7日を生き抜いた**達成**の音」。

差し替えは `DailyDerbyStage.tsx` の `VERDICT_SOUND` 表と `soundCatalog` を見ればよい。

## 実装の所在

- キュー定義: `apps/web/components/daily-derby/DailyDerbyStage.tsx`

  と `apps/web/lib/band-race.ts` (`ACT_VERDICT_AT` = 生死確定 55秒)
- 音源の尺を大きく変える場合は上記定数を再調整すること
