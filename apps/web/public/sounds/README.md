## 設計方針(2026-07-21・レビュー側判定)

音は一枚岩ではなく2種類に割れる。この区別が全判断の土台。

| 種別 | 例 | アート依存度 | 扱い |
|---|---|---|---|
| **イベント音**(情報) | 生存の合図、BURNの衝撃、幕の切替 | 低い | 足してよい |
| **ベッド音**(雰囲気・劇伴) | 群衆、緊張を煽る弦、競り合いのSE | 高い | 7/10 ドット走行と同じ轍 |

- **競り合いを煽る音は足さない。** 中間順位のデータは存在しないので、それは
  視覚で排除したフィクションを聴覚で再導入することになる
- **収支のプラス/マイナスで音を変えない。** パチンコの当たり音そのものであり
  R1レッドライン(射幸性を訴求しない)に触れる。価値は数字が語り、音は中立に保つ
- **音が無くても成立することが前提。** 多数がミュートで視聴する。音は加点であって前提ではない
- 帯レースの最良の音は、群衆を引いて作る**沈黙**である

## 発注待ち(オーナー支給WAV・2本)

**2026-07-21 オーナー指摘**: 生存 / DAY7チャンピオン / BURN後の形見ドロップ の
3つが、すべて `own-good.mp3` の同一音だった。感情が違うので分ける。

| 出来事 | 感情 | 現在 | 必要な音 |
|---|---|---|---|
| 生存 | 安堵「今夜も生き延びた」 | own-good | **そのまま** |
| DAY7チャンピオン | 祝祭 | own-good | **champion sting**(発注) |
| BURN後の形見ドロップ | 慰め | own-good | **memorial drop**(発注) |

| 音 | 尺 | 性格 | 注意 |
|---|---|---|---|
| champion sting | 2〜3s | 儀式的・荘厳。7日完走の**達成**を祝う | 「換金」でなく「達成」の音。金の当たり音にしない(R1) |
| memorial drop | 1〜1.5s | 静かで温かい。**慰め**。BURN音の1.6秒後に鳴る | 祝福・獲得の音にしない。馬を失った直後である |
| **settle-out** | 0.4〜0.6s | 厩舎から**出ていく**。下降・柔らかい | 「儲かった音」にしない。方向だけを伝える |
| **settle-in** | 0.4〜0.6s | 厩舎に**入ってくる**。上昇・柔らかい | 「当たり音」にしない。方向だけを伝える |

**方向音は配線済み。** `settle-out.mp3` / `settle-in.mp3` をこのフォルダに置けば
コード変更ゼロで鳴り始める(未配置の間は `playOneShot` が握りつぶすので無音のまま)。

★出ていく/入ってくる の**方向**だけを鳴らすこと。**収支のプラス/マイナスで音を
変えてはいけない** — それはパチンコの当たり音そのもので、R1レッドライン
(射幸性を訴求しない = AML/KYC不要見解の土台)に触れる。価値は数字が語る。

7日完走はゲーム経済の頂点(200 USDT買戻し)なので通常生存と同じ音は安売り。
形見ドロップは逆に、失った直後に喜びの音が鳴るのが不適切。

**専用音が届くまでは own-good を fallback として使う(無音にはしない)。**
差し替えは `DailyDerbyStage.tsx` の `VERDICT_SOUND` 表の右辺を変えるだけ。

## 実装の所在

- キュー定義: `apps/web/components/daily-derby/DailyDerbyStage.tsx`
  (一発モノ = `cues`、ループ = `windows`。`fresh` 窓で途中参加時の飛び越え再生を防ぐ)
- タイムライン定数: `apps/web/lib/daily-derby.ts` (`RACE_RUN` / `LOGS_FROM` / `COMPLETE_AT`)
  と `apps/web/lib/band-race.ts` (`ACT_VERDICT_AT` = 生死確定 55秒)
- 音源の尺を大きく変える場合は上記定数を再調整すること
