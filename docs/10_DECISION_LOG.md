006. Race seed uses Server Commit-Reveal in v1.0.
007. Each race has exactly one independent race_seed and seed_hash.
008. Race Participant Snapshot is created at Daily Batch start and is immutable.
009. Marketplace locks during Daily Race Batch.
010. Purchase Session uses immediate fund locking.

014. Horse current_day increments only after Race Survival.
015. AI Profit Taking creates market listing only and does not transfer ownership.

023. Ledger account structure includes user, reserve, settlement, deposit, and withdrawal clearing accounts.
024. Day0 Mint Reserve Allocation v1.0: Buyback 93.60, MLM 5.40, Operating 0.70, Emergency 0.30.
025. Economy Status is determined by policy thresholds, AI only recommends.
026. Economy Status uses Stability Rule and Emergency Lock Period.
027. Daily Settlement Batch order v1.0 is fixed by batch_algorithm_version.
028. Admin Recovery Procedure is required for failed/partial failed batches.
029. Database spec scope is major columns, constraints, indexes, RLS, immutability; not full DDL.
030. API spec scope is architecture-level; OpenAPI is generated by Claude Code.
031. Repository structure and execution boundary are fixed.
032. Definition of Done and Test Policy are fixed.

036. Daily Training v1.0 is a one-race modifier with snapshot cutoff, no permanent ability increase, and deterministic LUCK random range adjustment.
037. Deposit and Withdrawal v1.0 supports USDT only, defaults to Polygon PoS USDT, uses Ledger for all balance changes, and excludes multi-chain support.
