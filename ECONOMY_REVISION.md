# 経済改定 v1.1(Decision 069)— 改定理由と実証データ

> 2026-07-04 / 開発者がオーナー委任(RTP100%超の是正指示)に基づき実施。本文書は正式追認用の提示資料。
> 対応コミット: Decision 069一式 / 検証: `packages/settlement-engine/scripts/econ-experiment.mjs`(再実行可能)

## 1. なぜ変更が必要だったか(実測データ)

10万ユーザー×30日のフルシミュレーション(Completion Gate G10)と「需要急停止ドリル」で以下を実測:

| 実測項目 | 設計想定 | 実測値 |
|---|---|---|
| Day7到達率 | 47.8%(0.9^7) | **約49.4%**(floor丸め+Revenge Buff効果) |
| 1ミントあたりBuyback負債 | 95.66 | **約98.9 USDT** |
| 同・準備金積立 | 93.60 | 93.60(変わらず) |
| **総合RTP** | 想定: 運営利益1% | **約100.6%(プレイヤー側が+0.6%)** |

つまり旧設定は**構造的に約5.3 USDT/mintの持ち出し**で、新規ミントの流入(成長)がある間だけ支払いが回る状態だった。

**需要急停止ドリル(旧設定)**: 300購入/日×15日→需要ゼロ。
→ **停止9日後にBuyback支払いが枯渇。未払い279件**(検証ログ: 準備金5.14 USDTで停止)。
200 USDT保証への信頼はコミュニティ戦略の生命線であり、これは許容できない。

## 2. 変更内容(プレイヤーへの約束は不変)

**変えていないもの**: Buyback 200 USDT・7回払い・D+1開始 / 価格カーブ(100〜177.16)/ 購入ロック177.16 / レースの決定論・リプレイ検証。

| # | 変更 | 旧 | 新 |
|---|---|---|---|
| 1 | Burn率(Status連動) | 10.0/10.4/10.8/11.2% | **10.7/11.1/11.5/11.9%**(日次生存率90%→89.3%) |
| 2 | ミント手数料 | なし(100) | **+2 USDT(計102)**。1→運営、1→Buyback緩衝 |
| 3 | P2P手数料 | 0(憲法) | **売り手2%**(買い手は表示価格のまま)。1%→運営、1%→Buyback緩衝 |
| 4 | ミント担保ゲート | なし | **準備金が「全出走馬の到達見込み(実測補正+3.5%マージン)+支払予定残」を下回る日はミント停止**(決定論・エンジン強制) |

- 憲法改定: 「P2P fee is always 0」→「2%固定(ポリシーバージョン)」。**手数料変更APIの禁止は維持**(管理画面からも変更不可)
- ゲートが閉じた日の購入者は従来の「ミント上限到達」と同じく全額自動返金(Step 27)

## 3. 変更後の実証(同一ドリル・同一条件)

**結果: 40日間・支払い5,404件・未払いゼロで完走。**

| 観測 | 旧設定 | **v1.1** |
|---|---|---|
| 需要停止後の帰結 | 9日後に枯渇・未払い279件 | **全額支払い・未払い0件** |
| 全馬清算後の準備金 | −(枯渇) | **+866.73 USDT(黒字で閉幕)** |
| 運営利益(15日間の営業で) | 実質0 | **5,233 USDT(≈総流入の約2%)** |
| ゲートの挙動 | − | 13日目にカバレッジ0.997へ低下→**自動でミント停止**→負債解消とともに回復(最低0.992、未払いに至らず) |

補足: v1.1では総合RTPは約96〜97%となる(mint購入者のEV: 到達≈47%×200=94/102 + 転売オプション価値。「約半数が2倍水準に到達」という成功体験設計は維持)。

## 4. 数式サマリ

- 到達率(実測補正): 理論(1−burn)^7 × 約1.033(floor+Buff)。v1.1: 0.893^7×1.033 ≈ **46.8%**
- 1ミント収支: 流入 93.60+1.0(手数料緩衝)+P2P緩衝 ≈ 94.6+ vs 負債 ≈ 93.7 → **黒字化**
- ゲート係数: `MINT_COVERAGE_FACTOR_V1 = 1.035 × 0.893^r`(r=残レース数、8桁切上げ・決定論)

## 5. 差し戻し方法(オーナーが不採用とする場合)

`domain/constants.ts` の BURN_TARGET_RATE_V1 / DAY0_MINT_FEE / P2P_FEE_RATE と ledger movements の該当2関数、`liquidity_policy_v1.1` マイグレーションを戻す(全変更はDecision 069のコミットに集約)。ただしその場合、本文書1章の破綻シナリオが復活する。
