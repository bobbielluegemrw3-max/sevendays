# /admin 全面リデザイン — 実装仕様書（Ops Console）

対象リポジトリ: `bobbielluegemrw3-max/sevendays`（最新 main）
対象: 管理画面 **全10ビュー**（Dashboard / Economy / Users / Items / Races /
Support / Batches / Withdrawals / Recovery / Audit）が共有する
`apps/web/app/admin.module.css` と各ビューの `.tsx`。

## 0. 大方針（これだけは外さない）
- **ゲーム本編（ダッシュボード/レース/LP）のネオンは維持。/admin だけを「静かな運用コンソール」へ。**
- **機能・データ・API・ビジネスロジック・型は一切変更しない。** 変えるのは
  見た目（CSS）と一部の**マークアップ構造（div の羅列 → 本物の `<table>`）**のみ。
- ダーク基調とマゼンタ（=admin識別色）は残す。ただし色は**状態表現と操作面だけ**に使う。

## 1. 差し替えるファイル
| ファイル | 内容 |
|---|---|
| `apps/web/app/admin.module.css` | **全面差し替え。** 同梱のものをそのまま使用。全10ビューへ波及。 |
| `apps/web/components/AdminNav.tsx` | 絵文字廃止・現在地下線ハイライトの新ナビ（同梱）。href/順序は既存ルートに合わせる。 |

CSS のクラス名 API（`.wrap .ph .phTitle .statRow .stat .sec .menu .mCard
.tableWrap .tbl .num .mono .st(.stGood/.stWarn/.stBad/.stNeutral) .tag .btn
.controls .inp .sel .note .dossier .desktopTable .mcard`）を各ビューで使う。

## 2. ビュー横断の変換ルール
現行は行を `flex-wrap` の `div`（“カード”）で並べており**列が揃わない**のが最大の欠点。
一覧系はすべて**列見出し付きの `<table class="tbl">`** に置き換える。

- **数値・金額・USDT・日付** → `<td className={s.num}>`（右揃え・等幅・`tabular-nums`）。
  単位は `<span className="u">USDT</span>` で薄く。
- **ID/ハッシュ/アドレス** → `className={s.mono}`、長いものは `className={s.ell}` で省略。
- **状態** → `<span className={`${s.st} ${s.stGood}`}>` 等。色は good/warn/bad/neutral の4種のみ。
- **見出し（Orbitron）はページタイトルとブランドだけ**。データ本文は等幅（数値）＋日本語サンス（名称）。
- 装飾グラデ・ネオングロー・box-shadow のグローは**全廃**。境界は `1px var(--c-line)` 1本。

## 3. ビュー別の要点
- **Dashboard** — 上部に4連 `statRow`（状態は `statBig.ok/.warn/.bad` の左3pxバーで即判断）。
  下部の遷移メニューは `menu`＋`mCard`（絵文字を単色 `mGlyph` に置換）。
- **Batches / Withdrawals / Recovery** — `tbl` 化。金額右揃え、状態ピル、行末に `btn`/`btnDanger`。
- **Economy** — 勘定残高テーブル。残高は `num` 右揃えで縦比較可能に。
- **Users** — `controls`（`inp` 検索＋`sel` 絞り込み＋`cnt` 件数）→ `tbl` で走査 →
  行クリックで `dossier` 展開（台帳サブテーブルも `tbl` で整列）。
- **Races** — 開催回テーブル（日付・出走数・Burn数・状態）。
- **Support / Audit** — 時系列を等幅で縦走査。Audit は `mono` の時刻＋アクター＋アクション。

## 4. モバイル（必須）
- `tbl` は `.desktopTable` でラップし、**720px 以下で `display:none`**。
- 同じデータを `.mcard > .mc`（1件=1カード：`mcTop`＋`mcGrid`＋`mcCell`）で並列に用意し、
  720px 以下で表示。`statRow` は 2 列、`menu` は 1〜2 列へ自動リフロー。
- ナビ `.nav` は横スクロールの帯（`overflow-x:auto`、スクロールバー非表示）。

## 5. 視覚リファレンス
`Admin Console Proposal.html` を同梱。承認済みの最終見た目（NAV/Dashboard/
Batches/Economy/Users/Audit の代表画面＋モバイル）。**このHTMLの `<style>` が
`admin.module.css` の元**なので、迷ったらHTMLを正とすること。ブラウザで開けば
幅を狭めてモバイルのカード積みも確認できる。
