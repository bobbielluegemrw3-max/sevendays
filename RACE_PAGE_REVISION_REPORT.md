## 3. 審判リーチ演出（モニター内）

正典 `RACE_VERDICT_REACH_DESIGN_BRIEF.md` に沿い、デザイナーZIP（`レースページリーチ演出.zip`）を参考にしつつ実装。

### ★大前提（オーナー厳命）
**元の美しい盤 `BandRaceAct.tsx` ＋ `.br*` CSS（`daily-derby.module.css`）は一切変えない。** デザイナーが依頼範囲（①12種リーチ ②ログの動き）を超えて元の盤まで作り変えてしまった経緯があり、**演出は元の盤の上に薄く重ねるだけ**という制約を厳守。

### 実装（増分1 `3bcb955` → 増分2 `4d4aad1` → 完成 `cf4f174`）
- **`ReachFxLayer.tsx`**（新規・canvasオーバーレイ・`pointer-events:none`）をモックの `draw()` から移植し、`DailyDerbyStage` の帯領域に重ねる。空気のキーライト（danger基準で金→赤）＋ 破裂（fate＝金/赤の衝撃波）を**実状態で駆動**。
- **期待tier 6段**: bands API（`derby.ts` の entries クエリ）に `s.total_value` を追加（ADR-007＝所有者情報でない・レース前総合値）し、`BandEntryRow`→`BandsResponse`→`BandEntry`→`BandRaceFrame.myTotalValue` へ結線。自分の馬の総合値 → 激アツ≥78／あつい≥66／普通≥55／期待低≥45／期待薄 の tier 色を FX へ。
- **裏切りフリーズ（12種の骨格）**: `level = survive ? tierIdx : 4-tierIdx`、`level>=2`（期待と逆）で寸止めフリーズ（freezeMs 520/720/1000ms・§0-5準拠）。**7秒暗黒 tier5 は32秒の帯尺に収まらないため上限1.0s**。盤本体は不変のまま、ラップの `filter` でフリーズ＝無彩化／決着＝暗転、シネスコ黒帯を重ねる。
- テスト: `api.test.ts` の entry キー厳密チェックに `total_value` を追加（78 pass）。

### ★段階的ビープ音（`fd658cf`・オーナー支給5音源）
帯レース後半 **LINE 幕**で残りの他馬を一段ずつ出し切る各ステップに、期待tierの「ピ」を1回ずつ発火（`ピ…ピ…ピ` と視覚に同期）。**前半 REVEAL（88%を一気）は無音**。`public/sounds/reveal-tier0..4.wav`（激アツ/あつい/ふつう/期待低/期待薄・volume 0.8）。既存 `soundOn` トグルを尊重。**元の盤は不変・音を上に重ねただけ。**（オーナー鳴動確認済み）
