## Decision 101(草案): 総合値エンジン(レースエンジンV2)とレアリティ廃止

1. **スコア式(V2)**: `final_score = 総合値(0〜100) + 条件補正(±4) + 運(±3・Irwin-Hall)`
   - 条件補正 = 天候×馬場×コースへの「備え」(タイプ適性+レースアイテム)。commit-reveal・決定論・検証可能性は現行と同一
   - 切れ味は**(a)鋭いまま**(混合母集団で適当23.7% vs 完璧93.5%のチャンピオン率 — eスポーツ路線・オーナー決定 2026-07-17)
2. **新規発行馬の総合値**: 40〜75の一様分布(平均57.5)。90台は生まれでは出ない — 育成でのみ到達
3. **変動**: 調教・アイテムで上下(Decision 104)。**減衰 −2.0/レース**(手入れしないと下がる)。
   **ソフトキャップ85**(85超の上昇は半減 — 90台を特別に保つ)。0〜100にクランプ
4. **レアリティ(COMMON〜LEGENDARY)は廃止**: RARITY_PROBABILITY/MODIFIERテーブル削除。
   **馬タイプ(SPRINTER/POWER/BALANCED/ENDURANCE/LUCK)は存続**(育て方の個性・適性・隠れた好みの軸)。
   LUCKの調教時運レンジ拡大(−2〜+4)は存続
5. **調子・疲労は独立ステータスとして廃止**し、総合値の増減に内包(内部の状態変数は実装が決める。
   表示は総合値1本)
6. **正直表現の義務**: 「上手い人が勝つゲーム」をガイド・オンボーディング・CSナレッジに明記。
   安全圏バッジ(上位40%=SAFE/下位25%=RISK)は目安であり保証でないことを常に併記

## Decision 102(草案): 1日2レース(LV制)と開催時刻

1. **LV.1〜7**: 1レース生存でLV+1(DAY表記を全面置換)。LV7走破=チャンピオン。最速3.5日
2. **開催【オーナー最終決定 2026-07-17】: 朝 8:00 MYT(9:00 JST)/ 夜 20:00 MYT(21:00 JST)**
   — **夜は現行のまま変更なし**(レビュー第1回で「MYTの朝8時と夜8時」と確定。
   同日先行の「夜19:00 MYTへ前倒し」決定はこれで**上書き・廃止**)。
   時刻の切替作業は不要になり、V2は朝レースの追加のみ
3. **バッチ**: `batch_runs` は (batch_date, slot MORNING|NIGHT) キー。冪等キー・記録・台帳・my-results・
   透明性台帳・予報・ショーはすべてレース単位へ。日付境界はMYT深夜0時のまま
4. **価格表はLV読み替えで値を維持**(LV0=100 … LV6=177.16・買戻し200)。
   **チャンピオン報酬の分割は現行のまま「7回・日ごと」【オーナー決定 2026-07-17】**
   (完走が3.5日でも支払いは7日間に分散 — 買戻し準備金の資金効率が遥かに安全側)
5. **BURN率の器(8.0〜13.5%/レース)は不変** — 1日あたりの回転が2倍になるだけで、
   1ライン(7レース)の数学は不変(モデルシム検証済み・FUN_V2_PLAN §5.5.1 Q1)
6. **調教・利確・チケットは各レース前に1回**(1日2サイクル・§7合意)
7. 定常人口の目安: 出走1,000頭の維持に必要なミント≈392頭/日(シムQ2)— 広告費イベントと
   集客ペースの計画数値として記録
8. **買戻し準備金バックストップ(RTPシム突合 2026-07-17 の発見・§5.5.2)**: 買戻しの
   損益分岐(到達率46.8%)をジッターが上回るコホートでは準備金が枯渇し未払いが発生し得る
   (実エンジンで再現済み)。V2では**「PAY_DUE_BUYBACKS の直前に、準備金が当該バッチの
   予定支払い額を下回る場合、不足分を運営準備金→買戻準備金へ自動補填する」正規ステップ**を
   追加し、買戻し未払いを構造的にゼロにする(補填はLedger移動として透明に記録)

## Decision 103(草案): プール購入「◯◯$厩舎」(177.16ロックの廃止)

1. **予約の単位は「金額」**: パッケージバッジ 200$/500$/1000$(約7〜9頭)/2000$/5000$/10000$+自由入力。
   指定額を丸ごとロック。締切まで増減・取消自由
2. **割当**: 次に来るレース(朝・夜どちらでも)のバッチで、P2P出品(Decision 100の決定論抽選順)→
   LV0新規発行(102$)の順に**予算いっぱいまで割当**。**余りは必ず102$未満**まで使い切り自動返却
3. **予算超過の出品の扱い**: 買い手の残予算で買えない出品はスキップして**次の買い手のキューへ回す**
   (出品は誰かに買われる機会を失わない)。買い手はミントで埋める
4. **価格は固定価格表のまま**(動的価格は導入しない — 法務上の盾を維持・§10合意)
5. 旧: セッション単位の177.16ロック・同時セッション上限・auto_reserve は廃止し、プール型に統合。
   auto_reserve 相当は「毎レース、残高から◯◯$分を自動でプール予約」設定として再定義
6. ショーの個人結果に「YOUR NEW STABLE — 1000$が8頭になりました」(総合値付き披露)

## Decision 104(草案): 新調教システム(メニュー・隠れた好み・チケット)

1. **メニュー(6種・公開レンジ)** — 1レースサイクルにつき**2つまで組み合わせ**(同一メニュー2回も可):
   | キー | 名称 | 公開レンジ | 性格 |
   |---|---|---|---|
   | HILL | 坂路 | +1〜+5 | パワー寄り・外すと重い |
   | POOL | プール | 0〜+3(+疲労抜き) | 安全・下振れなし |
   | SPAR | 併せ馬 | −2〜+6 | ハイリスク実戦派 |
   | GATE | ゲート練習 | 0〜+4 | 出脚・堅実 |
   | WOOD | ウッドチップ | +1〜+4 | バランス |
   | REST | 休養 | 0(減衰を1回無効) | 手入れ・好み無関係 |
   ※レンジ値は実エンジンRTPシム突合(§2)後に最終確定。合計ロールがシムの
   informed(+3〜+8)/外れ(−4〜+2)の分布を再現するよう調整する
2. **隠れた好み(dna_hash決定論)**: 各馬は「大好物メニュー1つ(ロール上限側に寄る)」
   「苦手メニュー1つ(下限側に寄り、下振れ拡大)」を持つ。
   **法則**: 好みはタイプと相関(例: SPRINTERの70%は坂路かゲートが大好物)— タイプ別の
   傾向は攻略で学べるが、30%の例外は個体ごとの試行でしか分からない。
   **2連シナジー**: 大好物を含む組み合わせは追加ボーナス。
   全て後日reveal可能なコミット構造(検証可能・イカサマ反証可能)
3. **確定の瞬間にロール**(§8.2合意): プレビューは公開レンジまで。「これでいいですか?」で
   実数が確定しドラマチックに表示。スナップショット前ならやり直し自由・チケット/利確は初回のみ
   (A2で実装済みの流儀を継承)
4. **調教チケット**: 確定1回=1枚(A2実装済み)。用途はDecision 106(ジャックポット)
5. **有料アイテムの再編(2分類・§6.2/§8.3合意)**:
   - **調教アイテム**: 総合値に直接作用(大きめレンジ・下振れなし〜小)— 「化けさせる」課金
   - **レースアイテム**: 予報(的中70%)を読む「備え」— 当たれば+4〜+8相当の条件補正・
     外れれば−7〜−3。**「干渉・予想」ではなく「備え」**(公平性・R1維持)
   - **アイテムカタログは再マッピングではなく新規に作り直す【オーナー指示 2026-07-17】**:
     2分類(調教アイテム/レースアイテム)を前提にゼロから設計。既存35種は
     メインネットリセットで引退(テストネット資産のため補償問題なし)。
     新カタログ案はClaudeが起草→オーナー承認(アート発注も新規)
