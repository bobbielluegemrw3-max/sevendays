## 3. Claude側の仮説(2026-07-16時点・報告前の分析)

つまらなさの主因は法務制約そのものではなく、**表現とペーシング**にある可能性が高い:

| # | 仮説 | 中身 |
|---|---|---|
| H1 | **昼間は何も起きていないゲーム** | 心臓は20:00の約100秒だけ。日中に見せると「空のスタジアム」。セミナーが日中ならこれが直撃 |
| H2 | **初体験が遅すぎる** | 新規ユーザーがドキドキを味わうまで最大24時間。ゲームの初回セッションとして致命的 |
| H3 | **腕前の実感がない** | 調教1タップ+3〜5、あとはシード任せ。公正さの設計としては正しいが「自分が上手いから勝った」という物語が作れない |
| H4 | **馬が走らない** | レースなのに走る姿がない。前日に一般ユーザーからも同じ要望が出ていた(点はすでに2つある) |

重要な整理: **これらは経済・Ledger・法務整理を一切触らずに直せる層**(表現・演出・ペーシング・
オンボーディング)の問題。土台(毎晩確実に動き、1円も間違えず、誰でも検証できる金融エンジン)は
面白さの敵ではなく土台。捨てるものはない。足りない層が1枚ある。

## 6. オーナーの詳細報告(2026-07-17)— 生のフィードバックと triage

セミナーの結果を受けたオーナーの改修アイデア5本。原文の趣旨を保存し、Claudeの評価を併記する。

### 6.1 調教 — 「意味が全く分からない。あってもなくても無意味」
- **①利確調教**: レース後、全馬を調教しないと厩舎価値に馬の価格が反映されない → 毎日ログインの必然を作る
- **②調教チケット→週次ジャックポット**: 調教のたびにチケット、週1でジャックポット抽選
- **③調教の演出**: 今は「適用されました」だけ。絵も数値も変わらない
- 評価: ③は即やるべき(表現層・コスト小)。**調教の効果がレース結果のどこに効いたかの帰属表示**
  (「昨夜のスコア+5は調教分」)まで含めて「腕前の実感」を作る。①は罰則型より報酬型
  (調教済みだとボーナス)への転換を推奨 — 実マネーのゲームで「ログインしないと損」は怒りを生む。
  ②は**弁護士確認必須**(懸賞/抽選)。ただし調教=無料行為由来+原資=運営広告費なら
  「賭博」でなく「販促キャンペーン」として整理できる可能性が高い(092/095の同族)

### 6.2 レアリティ廃止 → 総合値0〜100 一本化
- 新規馬=0〜100の総合値。調教とアイテムで90に育てると勝ちやすい。「納得して挑む」を作る
- アイテムは2種に再編: **調教アイテム**(総合値に作用)/**レースアイテム**(天候・馬場・コースに関与)
- 評価: **可読性の勝利。方向性に強く賛成**。1つの数字は「70を88に育てた」という腕前の物語を作る。
  ただし2つの正直な注意:
  1. **BURN枠は頭数比で固定(憲法8.0〜13.5%)なので相対戦** — 全員が90になっても8%は消える。
     「90なら勝てる」ではなく「総合値が高いほど生存確率が上がる」が正直な設計・表現
  2. レースアイテムの「天候や馬場に**干渉**」は公平性の根幹(誰も結果に介入できない・
     commit-reveal)と衝突し、「**予想**」はR1レッドラインに接触する。
     → 「条件への**備え**」(適性ブースト=既存アイテムの思想)として設計し直すのが筋
- 実装レイヤ: **表示先行が可能** — 既存の能力+調子+疲労+レアリティ+調教から合成した
  総合値を先に「見せる」(エンジン不変・即日可)。機構として採用するのはレースエンジンV2
  +BURN分布シミュレーション+メインネットリセット(C層)

### 6.3 レース回数 — 「1日1回は退屈すぎる。初期は朝9時+夜21時の2回に」
- 評価: フル2回/日は価格ラダー・疲労・「7日間」の物語・batchDateFor・スケジューラー全てに波及
  = **メインネットリセット級(C層)**。中間案として**朝のエキシビションレース**(経済に触れない
  見せ場・賞品は運営広告費枠 or コスメティック)ならB層で先行可能

### 6.4 決定論と「吐き出し」— 運営広告費でユーザーに勝ちを体感させる時期が必要
- 評価: 賛成。決定論・憲法(BURN率・価格表)は**不変のまま「足す」**設計にする:
  Ledgerに**運営広告費(MARKETING_BUDGET)口座**を新設し、ジャックポット・イベント賞金・
  ボーナスの原資として台帳上で明示的に払い出す(透明性台帳にもそのまま載る=正直)。
  パチンコの「出す時期」に相当する裁量は、この枠の予算額として運営が握る。**弁護士確認は必要**

### 6.5 177$ロック撤廃 — 「1000$入れた人がほとんど馬を買えた感覚にならない」
- 現行: 予約1件につき PURCHASE_LOCK_AMOUNT=177.16(割当されうる最高日齢の価格)をロック
  → 1000$で5枠。「買えた感覚」が薄い
- オーナー案: 締切時刻までのウォレット残高総額に対して決定論的に価格・割当を決める
- 評価: **動的価格(残高総額→価格決定)は価格表の固定性を壊し、法務リスクをむしろ上げる**
  (投機商品化)。ただし課題認識は正しい。**価格表を固定したまま体験を変える中間案**:
  - **プール型予約**: 「この1000$で厩舎をつくる」— 指定額をまるごとロックし、割当エンジンが
    その予算で買えるだけ割り当てる(5〜9頭・余りは自動返還)。決定論・価格表・公平性は不変
  - 既存の auto_reserve_max=MAX(残高いっぱい自動予約)が思想的な先行実装 — 見せ方が
    「設定の隅」に埋まっているのが問題で、購入体験の主役に昇格させる

### 6.6 triage(層別)

| 層 | 内容 | 前提 |
|---|---|---|
| **A: 即実装可(経済・法務不変)** | 調教③演出+効果の帰属表示 / 総合値0-100の**表示先行** / プール型予約の**UX前段**(MAX予約の主役化・ロックの見せ方) | オーナーGOのみ |
| **B: 小さめの経済/機能追加** | 運営広告費口座(Ledger) / 調教チケット+週次ジャックポット / 朝のエキシビションレース / プール型予約の本実装 | Decision+弁護士確認(②④)+シミュレーション |
| **C: 大改修(メインネットリセット級)** | レアリティ廃止→総合値の**機構**採用(レースエンジンV2) / 1日2レース / 調教①(価値反映ゲート) | V2定数+BURN分布シミュレーション+リプレイ互換+リセット計画 |

## 7. 【合意済み】調教システム改修(2026-07-17 オーナー承認)

議論で確定した設計。オーナーの原文4点(①無料アクション ②メニュー増 ③複数ボタンの
組み合わせ+最後に「これでいいですか?」④確定1回でチケット+前レースの利確が確定)を
以下のループに落とし、オーナーが「合っています」+確認3点すべてOKと承認した。

### 7.1 新しい毎日のループ(1頭あたり)

1. 日中、厩舎を開くと昨夜生き残った馬に「**未回収 +12.10$**」と「調教未確定」マーク
2. 馬を開くと**総合値**(例: 72)が大きく表示。調教メニュー(複数ボタン)を組み合わせると
   「今夜の総合値 72 → 76」のプレビューがリアルタイムに動く
3. **「これでいいですか?」→ 確定**で3つが同時に起きる:
   (a) コイン演出とともに未回収額が総資産へ加算(**利確**)
   (b) **調教チケット+1枚**
   (c) 総合値バーがアニメーションで伸びる
4. 20:00まで**調教のやり直しは何度でも自由**。ただしチケットと利確は**初回確定の1回だけ**
5. **有料の調教アイテム**は総合値が**大きく**動き、**馬の見た目が変わる**(装具・オーラ・色)。
   無料調教は数値のみ・見た目そのまま(オーナー明示の区分け)
6. レース後の結果に**帰属表示**: 「あなたの調教が+4を生んだ — 調教なしなら順位◯位下だった」
7. **週1回、20:00ショーの最終幕でジャックポット抽選**: 賞金USDT・チケット枚数が多いほど
   当たりやすい(**大厩舎有利=オーナー承認**)・原資は運営広告費・commit-reveal検証可能

### 7.2 利確の意味(解釈の確定)

「利確」は**解釈A=収穫の儀式**で確定: 昨夜の値上がりは調教確定するまで厩舎評価額・総資産に
**表示されない**(未回収として見える)。経済の実体(売却価格・買戻し)は変えない。
罰ではなく「刈り取る快感」で毎日ログインさせる。

### 7.3 実装の2段構え(オーナー承認済み)

- **先行(A層・今すぐ・経済不変)**: 総合値の表示 / 未回収マークと利確演出 / 確定ボタン /
  チケット付与と枚数表示(**抽選はまだ**)/ 帰属表示 / 調教リアクション演出。
  調教の中身は現行3種のまま・**見せる数字はすべて実定数**(見かけだけのボタンは作らない —
  それは「意味のない調教」の再生産だから)
- **完成形(C層・エンジンV2・メインネットリセット時)**: 調教メニュー本格拡張+組み合わせ
  (競馬らしい語彙: 坂路・プール・併せ馬・ゲート練習等 — **メニュー案はClaudeが作る**・承認済み)
  +総合値が本当の計算機構になる+調教アイテム再編
- **ジャックポット抽選の本稼働のみ弁護士回答待ち**(チケットを貯める部分は先行可)
- **「全馬まとめて確定」は維持**(エースだけ手動・残り一括の現行思想 — 100頭勢対応・承認済み)
